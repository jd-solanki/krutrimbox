# Derive implementation sequence from issue numbers

krutrimbox discovers Implementation Issues from a PRD's native GitHub sub-issue links rather than by parsing issue bodies. It fetches the PRD's attached sub-issues directly (GitHub returns them as part of the PRD), and each Implementation Issue confirms its PRD through GitHub's native parent link (`parent_issue_url`), not a `## Parent` body section. It derives the Implementation Sequence by sorting those Implementation Issues by GitHub issue number, so the issue publishing workflow must create sub-issues in dependency order and treat issue number order as the durable execution order.
